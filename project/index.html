$ Project - \infty $：更好的 CAE 、更好的服务、更好的制造业

  • 只是自己先想想
  • 供给：更好的 CAE
  • 需求：更好的制造业
  • 对接手段：更好的服务

1 现状与特点之拙见 - CAE：开发者、软件、硬件

1.1 开发者

  • 厂商团队
    • 以客户需求与技术发展为导向
    • 相对完善的开发基础与框架
  • 高校团队
    • 以发文章做项目为导向，提出原理、改进方法、拓展应用为主
    • 缺乏开发的基础平台
  • 非厂商非高校团队
    • 以客户要求为导向，满足客户要求为主

1.2 软件

  • 商业软件
    • 销售 License 为主，价格相对高，但有破解版
    • 有售后
    • 接口有限、相对封闭
    • 资料多、易操作、易上手
  • 开源软件
    • 获取和使用免费，部署在超算相对容易
    • 自己对结果负责
    • 用户自由度高
    • 需要用户有一定水平
  • In-House
    • 非公开
    • 以提高效率或满足特定需求为目的
    • 自编程或二次开发

1.3 硬件

  • 个人计算机
    • 用于学习
  • 工作站
    • 现阶段的主要设备
    • 数据输入及输出可在企业或机构内部完成
  • 超算
    • 用于大规模问题
    • 需考虑数据传输与安全

2 现状与特点之拙见 - 制造业：使用者、企业/机构、行业

2.1 使用者

  • 工程师
    • 目的与需求明确
    • 能够基于专业基础对结果进行一定程度的判断
  • 教授/教师
    • 熟悉领域内的基本原理
    • 对 CAE 的使用可能并不熟悉
  • 学生
    • 未来的工程师
    • CAE 使用水平取决于研究方向、知识积累、个人投入

2.2 企业/机构

  • 大企业
    • 资金、人才充足
    • 惯性大
    • 要求高
  • 中小企业
    • 需求相对常规
    • 重视成本与效率
  • 高校
    • 有一定技术积累
    • 需求相对前沿、需要开放接口

2.3 行业

  • CAE 使用规范
    • 企业内部大多会制定 CAE 的使用规范，将经验固化
    • 行业规范的制定要困难得多
  • CAE 评价体系
    • 目前似乎并没有一套评价 CAE 软件或解决方案优劣的体系及做法
    • 主要仍依靠行业内的应用效果、使用经验、推广程度
    • 缺乏评价体系，对国产 CAE 软件的发展不利

3 现状与特点之拙见 - 服务：线上、线下

3.1 线上

  • 公众号、微博
    • 资讯与宣传
  • 博客、Wiki
    • 介绍知识
  • 网站
    • 视频教程
    • 在线接入计算资源
  • 论坛、群聊、邮件
    • 交流讨论问题

3.2 线下

  • 销售
    • 一定程度上靠积累、经营人脉
  • 培训
    • 以厂商、销售方提供培训为主
  • 咨询
    • 功能的实现
    • 解决方案
    • 二次开发
  • 直接参与项目
    • 作为分包商承接项目

4 更好的 CAE：方向

4.1 分析更准

  • 开发者、使用者：水平提高
  • 软件：算法改进
  • 硬件：精度提高

4.2 功能更强

  • 软件：适用范围扩大，扩展性增强，稳定性更好

4.3 操作更易

  • 软件：界面更加友好，使用者的意图更加容易实现

4.4 耗时更短

  • 软件：算法改进，分布式计算能力增强
  • 硬件：运算速度提升，传输速度提升

5 更好的制造业：目标

5.1 高质量

  • 准确预测产品性能，从而能够改进设计，提高质量

5.2 高效率

  • 分析、设计周期大幅缩短，能够灵活应对外部条件与需求的变化

5.3 低成本

  • 设计成本、试验成本、时间成本，得到有效控制

6 更好的服务：作为

6.1 拉近制造业与 CAE 的距离

  • 培养用户、发展客户
  • 细分行业、聚焦需求、展示方案
  • 在线提供对商业软件、开源软件的技术支持，降低技术门槛
  • 提高CAE 的可达性，更多利用公共计算资源，降低设备门槛
  • 改善CAE 的收费方式，降低价格门槛

6.2 让制造业用到好的 CAE

  • 提供客观的技术资讯与新近的学术进展
  • 实现使用者与开发者的双向对接
  • 解决方案可靠、可定制、可增强
  • 促进优秀的 In-House 进入市场

6.3 让制造业用好 CAE

  • 提高使用者水平
  • 参与制定使用规范
  • 参与建立评价标准
