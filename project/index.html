$ Project - \infty $：让CAE 更好地服务于制造业

- CAE

- 软件

  • 商业软件
    • 质量相对有保证
    • 学习资料相对多
    • 易操作、易上手
    • 存在破解版
    • 销售 License 为主
    • 价格相对高
    • 接口有限、相对封闭
  • 开源软件
    • 免费
    • 部署在超算相对容易
    • 用户自由度高
    • 对用户要求高
  • In-House
    • 非公开
    • 以提高效率、满足特定需求为目的
    • 自编程、二次开发

- 硬件

  • 个人计算机
    • 学习
  • 工作站
    • 主力
    • 数据输入及输出都可在企业内部完成，安全有保证
  • 超算
    • 大规模问题
    • 需要考虑数据传输与安全

- 开发者

  • 厂商
    • 客户需求与技术发展是驱动力
    • 开发的基础、框架完善
  • 高校
    • 发文章导向，以提出原理、改进方法、拓展应用为主
    • 缺乏基础性平台
  • 个人/团队
    • 客户导向，以满足客户需求为主
    • 盈利不够稳定

- 服务

- 线上

  • 形式
    • 宣传、讨论、资讯、教程、知识、计算
  • 媒介
    • 公众号、微博：适合宣传
    • 博客、Wiki：适合知识的展示
    • 网站：适合资讯、教程的展示
    • 论坛、群聊、邮件：讨论为主
    • SaaS 平台：在线接入计算资源

- 线下

  • 销售
    • 一定程度上靠积累、经营人脉
  • 培训
    • 以厂商、销售方提供培训为主
  • 咨询
    • 功能的可实现性
  • 提供解决方案
    • 软、硬件的组合
    • 二次开发
  • 直接参与项目
    • 利用知识、经验、熟练程度的优势参与项目

- 制造业

- 行业

  • CAE 使用规范
    • 企业内部大多会制定 CAE 的使用规范，将经验固化
    • 行业规范的制定要困难得多
    • 核心企业的规范向承包商扩散、龙头企业的规范向全行业扩散，会带动行业水平的提升
  • CAE 评价体系
    • 目前似乎并没有一套评价 CAE 软件或解决方案优劣的体系
    • 主要依靠行业内的应用效果、使用经验、推广程度
    • 缺乏评价体系，对国产 CAE 软件的发展不利

- 企业

  • 大企业
    • 资金、人才相对充足
    • 惯性大
    • 要求高
  • 中小企业
    • 看重成本与效率
    • 需求相对常规
  • 高校
    • 需要接口以对接方法的改进或创新
    • 需求相对前沿
    • 有一定技术积累

- 使用者

  • 工程师
    • 目的、需求明确
    • 能够基于专业基础对结果进行一定程度的判断
  • 教授/教师
    • 熟悉领域内的基本原理
    • 对 CAE 的使用可能并不熟悉
  • 学生
    • 未来的工程师
    • CAE 使用水平取决于知识积累、个人投入、研究方向

- 更好

- 分析更准

- 使用更广

- 耗时更短

- 成本更低
